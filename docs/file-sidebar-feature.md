# 文件侧边栏功能说明

## 功能概述

已成功实现类似参考图片中的文件侧边栏和可调整大小的预览窗口功能。

## 主要特性

### 1. 右侧文件栏
- **位置**：位于界面右侧，默认宽度320px
- **内容**：显示工作区中的所有文件和目录
- **排序**：目录优先，按字母顺序排列
- **图标**：使用文件夹/文件图标区分类型

### 2. 可拖动分隔条
- **位置**：聊天区域和文件栏之间
- **功能**：鼠标悬停时变为可拖动状态（col-resize光标）
- **拖动范围**：最小200px，最大800px
- **视觉反馈**：悬停时高亮显示为蓝色
- **线性响应**：鼠标移动距离与宽度变化1:1对应，提供流畅的拖拽体验
- **方向逻辑**：向左拖动增加侧边栏宽度，向右拖动减少宽度

### 3. 文件列表
- **点击文件**：选中文件并显示在预览窗口中
- **选中状态**：左侧有蓝色边框标识
- **目录支持**：点击目录可以展开/折叠（待实现）
- **刷新按钮**：右上角刷新图标可重新加载文件列表

### 4. 文件预览窗口
- **触发条件**：点击任意文件后自动展开
- **位置**：文件列表下方，占据侧边栏50%高度
- **内容**：以等宽字体显示文件文本内容
- **关闭按钮**：右上角×按钮可关闭预览
- **加载状态**：显示旋转动画和"Loading..."提示

### 5. 侧边栏折叠
- **折叠按钮**：左上角箭头按钮可折叠/展开整个侧边栏
- **状态保持**：折叠后仍保留分隔条，可重新展开

## 技术实现

### 前端 (React)
- **状态管理**：使用useState管理侧边栏可见性、选中文件、预览内容等
- **拖拽逻辑**：通过鼠标事件监听实现分隔条拖动
- **API调用**：fetchFileList()获取文件列表，previewFile()获取文件内容

### 后端 (Node.js + HTTP)
- **新增API**：`GET /api/file-content?path=<file_path>`
- **安全机制**：路径验证确保只能访问工作区内的文件
- **错误处理**：返回JSON格式的错误信息

### CSS样式
- **Flexbox布局**：使用flex布局实现左右分栏
- **过渡动画**：平滑的宽度和背景色变化
- **响应式设计**：适配不同屏幕尺寸

## 使用方法

1. **查看文件列表**：启动应用后，右侧会自动显示文件列表
2. **调整侧边栏宽度**：将鼠标移到分隔条上，按住左键左右拖动
3. **预览文件**：点击任意文件，下方会自动打开预览窗口显示内容
4. **关闭预览**：点击预览窗口右上角的×按钮
5. **折叠侧边栏**：点击侧边栏左上角的箭头按钮
6. **刷新文件列表**：点击右上角的刷新图标

## 注意事项

- 目前仅支持文本文件的预览
- 二进制文件会显示为乱码或空内容
- 大文件可能需要较长的加载时间
- 文件路径相对于工作区根目录
